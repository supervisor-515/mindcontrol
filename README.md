# 반증 장부 (Counter-Ledger)

> 머리 밖에 두는 **정직한 회계**. 작고 되돌릴 수 있는 선택을 *정하기 전에* "믿음의 예측"을 잠가서 기록하고, 며칠 뒤 실제 결과와 대조해 **빗나간 예측을 누적**하며, 결과를 깎아내리는 **재분류를 붙잡아 글로 기각**하는 단일 파일 웹앱.

설치 가능한 PWA(Progressive Web App)입니다. 브라우저에서 열어 그대로 쓰거나, 홈 화면에 설치해 오프라인 앱처럼 쓸 수 있습니다.

## 핵심 원칙 (무결성 규칙)

1. **예측 선잠금** — 예측과 확신도는 결과를 적기 *전에* 기록되고, 이후 읽기 전용으로 잠긴다.
2. **반증 누적 점수판** — 빗나간 예측이 영구히 쌓인다. "평균 예측 확신 vs 실제 적중률"의 보정 격차가 핵심 지표.
3. **재분류 포착·기각** — 결과가 괜찮을 때 깎아내리는 생각을 적게 하고, "기각" 버튼으로 명시적으로 무효화·기록한다.
4. **무손실 영속성** — 어떤 것도 조용히 사라지지 않는다. 모든 기록은 이 기기의 `localStorage`에만 저장되며, JSON 내보내기/가져오기로 백업한다.

## 파일 구성

| 파일 | 역할 |
| --- | --- |
| `index.html` | 앱 전체 (바닐라 JS, 단일 파일, 빌드 없음) |
| `manifest.webmanifest` | PWA 매니페스트 (설치 정보·아이콘) |
| `sw.js` | 서비스워커 (오프라인 캐시) |
| `icon.svg` / `icon-*.png` / `apple-touch-icon.png` | 앱 아이콘 (펼친 장부 + 믿음/실제 대조) |

## 로컬 실행

서비스워커는 `file://`에서 동작하지 않으므로 간단한 정적 서버로 엽니다.

```bash
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 열기
```

## GitHub Pages 배포 (웹앱 설치)

방법 A — **브랜치에서 배포** (가장 간단):

1. GitHub 저장소 **Settings → Pages**
2. **Build and deployment → Source**를 `Deploy from a branch`로
3. Branch는 `main`(또는 배포할 브랜치), 폴더는 `/ (root)` 선택 후 Save
4. 잠시 후 `https://<사용자>.github.io/mindcontrol/` 에서 열림

방법 B — **GitHub Actions** (이 저장소에 포함됨):

- `.github/workflows/pages.yml`이 `main`에 push되면 자동 배포합니다.
- 이 경우 Settings → Pages의 Source를 `GitHub Actions`로 설정하세요.

배포된 주소(HTTPS)로 접속하면 서비스워커와 매니페스트가 동작해 **설치 가능한 PWA**가 됩니다.

## 홈 화면에 설치하기

- **Android / 데스크톱 Chrome·Edge**: 주소창의 설치 아이콘, 또는 앱 우상단의 **설치** 버튼.
- **iOS Safari**: 공유 → **홈 화면에 추가**.

설치하면 전체화면 독립 앱으로 실행되고, 오프라인에서도 열립니다. 데이터는 기기에만 저장됩니다.
